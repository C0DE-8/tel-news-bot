# Talend API Tester collection notes

Live base URL:

```text
https://tel-news-bot-ivory.vercel.app
```

For JSON requests also add:

```text
Content-Type: application/json
```

Admin/test/webhook-management routes use `TELEGRAM_ADMIN_CHAT_IDS` from the server environment. You do not add that value as a Talend header.
Scheduled posting on Vercel uses `/cron/post-news`. If you set `CRON_SECRET` on the server, pass the same value as `?secret=...` or `x-cron-secret`; if you leave `CRON_SECRET` empty, no cron header is needed.

On Vercel, also set this environment variable so the bot always knows which channel to post into:

```text
TELEGRAM_GROUP_CHAT_IDS=-1002195390106|Liberty forward
```

Use the real Telegram channel id. Channel ids are usually negative numbers.

## 2. Set Telegram Webhook

Purpose: Tell Telegram to send bot updates to your live Vercel API.

```text
Method: POST
URL: https://tel-news-bot-ivory.vercel.app/webhook/telegram/set
Headers:
  Content-Type: application/json
Body:
```

```json
{
  "url": "https://tel-news-bot-ivory.vercel.app/webhook/telegram",
  "allowedUpdates": [
    "message",
    "channel_post",
    "callback_query",
    "my_chat_member"
  ],
  "dropPendingUpdates": true
}
```

Use `dropPendingUpdates: true` after a webhook error like `500 Internal Server Error`. That clears Telegram's retry queue so old failed updates do not keep replaying. Keep `channel_post` in `allowedUpdates` so the bot can read channel posts.

## 5. List Known Groups/Channels

Purpose: See groups/channels the bot has learned from Telegram updates.

```text
Method: GET
URL: https://tel-news-bot-ivory.vercel.app/admin/groups
Headers: none
Body: none
```

Important: Telegram does not let bots list every group/channel they are in. The bot learns a channel after it receives `my_chat_member` from Telegram. Keep `TELEGRAM_GROUP_CHAT_IDS=-1002195390106|Liberty forward` on Vercel so the channel always appears.

## 14. Run Scheduled Post Check

Purpose: This is the route Vercel Cron calls. Use it in Talend to verify scheduled posting. It checks every saved group/channel and only posts when the saved interval is due.

Important: Vercel/serverless does not keep `setInterval` running after a request ends. This route is what makes scheduled posting work on the live URL.
Use the `schedules` object in the response to see `nextPostAt`, `nextPostInSeconds`, and `countdown`.

```text
Method: GET
URL: https://tel-news-bot-ivory.vercel.app/cron/post-news
Headers: none
Body: none
```

Expected shape:

```json
{
  "ok": true,
  "checkedAt": "2026-07-04T00:00:00.000Z",
  "checked": 1,
  "posted": 0,
  "skipped": 1,
  "schedules": {
    "-1002195390106": {
      "enabled": true,
      "due": false,
      "reason": "interval_not_due",
      "intervalMinutes": 15,
      "nextPostAt": "2026-07-08T12:15:00.000Z",
      "nextPostInSeconds": 600,
      "countdown": "10m 0s",
      "postsSent": 0,
      "postLimit": 10,
      "postsRemaining": 10
    }
  },
  "errors": []
}
```

If you set `CRON_SECRET`, use:

```text
URL: https://tel-news-bot-ivory.vercel.app/cron/post-news?secret=YOUR_SECRET
```

## Recommended Start Order

1. `GET /health`
2. `POST /webhook/telegram/set`
3. `GET /webhook/telegram/info`
4. Add bot to your Telegram channel and make it admin
5. Wait for Telegram `my_chat_member`, or keep the channel in `TELEGRAM_GROUP_CHAT_IDS`
6. `GET /admin/groups`
7. `POST /admin/group-check`
8. `POST /admin/group-test-message`
9. `POST /admin/news-config`
10. `GET /cron/post-news`
