# Talend API Tester collection notes

Live base URL:

```text
https://tel-news-bot-ivory.vercel.app
```

For JSON requests also add:

```text
Content-Type: application/json
```

Admin/test/webhook-management routes use `TELEGRAM_ADMIN_CHAT_IDS` from the server environment. You do not add that value as a Talend header.
Scheduled posting is request-driven from saved SQL config. There is no cron route. Use `POST /admin/news-due` to make the server check SQL and post due news.

On Vercel, also set this environment variable so the bot always knows which channel to post into:

```text
TELEGRAM_GROUP_CHAT_IDS=-1002195390106|Liberty forward,-1003946270146|Liberty forward Trump Stock,-1004313221663|Liberty forward Grant
```

Use the real Telegram channel id. Channel ids are usually negative numbers.

Database note: the bot now saves to real SQL tables, not the old `tel_news_data.data_json` rows. In phpMyAdmin, check:

```text
tel_news_groups
tel_news_chats
tel_news_posted
```

Multi-channel posting is saved as one row per chat in `tel_news_groups`. Duplicate prevention is per chat in `tel_news_posted`; if a chat already has the same normalized title or URL fingerprint, the bot skips that article instead of sending it again.

Run `backend/migrations/002_create_normalized_news_tables.sql` manually only if you want to create the tables before the app auto-creates them.

In phpMyAdmin, check `tel_news_storage_checks`. If this table gets a row, database writes are working.

## 4. Set Telegram Webhook

Purpose: Tell Telegram to send bot updates to your live Vercel API.

```text
Method: POST
URL: https://tel-news-bot-ivory.vercel.app/webhook/telegram/set
Headers:
  Content-Type: application/json
Body:
```

```json
{
  "url": "https://tel-news-bot-ivory.vercel.app/webhook/telegram",
  "allowedUpdates": [
    "message",
    "channel_post",
    "callback_query",
    "my_chat_member"
  ],
  "dropPendingUpdates": true
}
```

Use `dropPendingUpdates: true` after a webhook error like `500 Internal Server Error`. That clears Telegram's retry queue so old failed updates do not keep replaying. Keep `channel_post` in `allowedUpdates` so the bot can read channel posts.

## 7. List Known Groups/Channels

Purpose: See groups/channels saved in the SQL `tel_news_chats` registry. On startup, the bot saves `TELEGRAM_GROUP_CHAT_IDS` into this table. It also saves chats it learns from Telegram updates.

```text
Method: GET
URL: https://tel-news-bot-ivory.vercel.app/admin/groups
Headers: none
Body: none
```

To verify whether the bot can still access/post to known chats:

```text
Method: GET
URL: https://tel-news-bot-ivory.vercel.app/admin/groups?check=true
Headers: none
Body: none
```

Important: Telegram does not let bots list every group/channel they are in. The bot can show only chats it knows from SQL, `TELEGRAM_GROUP_CHAT_IDS`, or incoming Telegram updates.

## Recommended Start Order

1. `GET /health`
2. `POST /webhook/telegram/set`
3. `GET /webhook/telegram/info`
4. Add bot to your Telegram channel and make it admin
5. Wait for Telegram `my_chat_member`, keep the channel in `TELEGRAM_GROUP_CHAT_IDS`, or `POST /admin/groups`
6. `GET /admin/groups`
7. `GET /admin/groups?check=true`
8. `POST /admin/group-check`
9. `POST /admin/group-test-message`
10. `POST /admin/news-config`
11. `POST /admin/news-due`
12. `GET /admin/news-config/-1002195390106`
