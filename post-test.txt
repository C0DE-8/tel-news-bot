# Talend API Tester collection notes

Live base URL:

```text
https://tel-news-bot-ivory.vercel.app
```

For JSON requests also add:

```text
Content-Type: application/json
```

Admin/test/webhook-management routes use `TELEGRAM_ADMIN_CHAT_IDS` from the server environment. You do not add that value as a Talend header.
Scheduled posting uses in-app timers from saved SQL config. There is no cron route.

On Vercel, also set this environment variable so the bot always knows which channel to post into:

```text
TELEGRAM_GROUP_CHAT_IDS=-1002195390106|Liberty forward
```

Use the real Telegram channel id. Channel ids are usually negative numbers.

Database note: the bot now saves to real SQL tables, not the old `tel_news_data.data_json` rows. In phpMyAdmin, check:

```text
tel_news_groups
tel_news_chats
tel_news_posted
```

Run `backend/migrations/002_create_normalized_news_tables.sql` manually only if you want to create the tables before the app auto-creates them.

```text

  Content-Type: application/json

```

Expected:

```json
  },

  "counts": {

    "posted": 0,

```

In phpMyAdmin, check `tel_news_storage_checks`. If this table gets a row, database writes are working.

## 4. Set Telegram Webhook

Purpose: Tell Telegram to send bot updates to your live Vercel API.

```text
Method: POST
URL: https://tel-news-bot-ivory.vercel.app/webhook/telegram/set
Headers:
  Content-Type: application/json
Body:
```

```json
{
  "url": "https://tel-news-bot-ivory.vercel.app/webhook/telegram",
  "allowedUpdates": [
    "message",
    "channel_post",
    "callback_query",
    "my_chat_member"
  ],
  "dropPendingUpdates": true
}
```

Use `dropPendingUpdates: true` after a webhook error like `500 Internal Server Error`. That clears Telegram's retry queue so old failed updates do not keep replaying. Keep `channel_post` in `allowedUpdates` so the bot can read channel posts.

## 13. Configure News Posting

Purpose: Configure the channel to receive news over time.

Important: `chatId` must be the channel id, not your private Telegram user/admin id.

```text
Method: POST
URL: https://tel-news-bot-ivory.vercel.app/admin/news-config
Headers:
  Content-Type: application/json
Body:
```

```json
{
  "chatId": "-1002195390106",
  "topic": "crypto",
  "intervalMinutes": 30,
  "postLimit": 10,
  "postNow": false
}
```

Allowed topics:

```text
crypto
politics
```

## 14. Configure Multiple Chats

Purpose: Configure the same news settings for more than one group/channel.

Use `chatIds` as an array:

```text
Method: POST
URL: https://tel-news-bot-ivory.vercel.app/admin/news-config
Headers:
  Content-Type: application/json
Body:
```

```json
{
  "chatIds": [
    "-1002195390106",
    "-1000000000002"
  ],
  "topic": "crypto",
  "intervalMinutes": 15,
  "postLimit": 10,
  "postNow": false
}
```

Or use comma-separated ids:

```json
{
  "chatIds": "-1002195390106,-1000000000002",
  "topic": "crypto",
  "intervalMinutes": 15,
  "postLimit": 10,
  "postNow": false
}
```

Expected result:

```json
{
  "ok": true,
  "groups": {
    "-1002195390106": {
      "topic": "crypto",
      "intervalMinutes": 15,
      "postLimit": 10
    },
    "-1000000000002": {
      "topic": "crypto",
      "intervalMinutes": 15,
      "postLimit": 10
    }
  }
}
```

```text
Headers:
  Content-Type: application/json
```

## Recommended Start Order

1. `GET /health`
2. `POST /webhook/telegram/set`
3. `GET /webhook/telegram/info`
4. Add bot to your Telegram channel and make it admin
5. Wait for Telegram `my_chat_member`, or keep the channel in `TELEGRAM_GROUP_CHAT_IDS`
6. `GET /admin/groups`
7. `POST /admin/group-check`
8. `POST /admin/group-test-message`
9. `POST /admin/news-config`
10. `GET /admin/news-config/-1002195390106`
